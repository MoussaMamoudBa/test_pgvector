Demo pgvector + FastAPI + Docker Compose

Exemple minimal pour tester la recherche vectorielle avec PostgreSQL (pgvector) et une API FastAPI, orchestrés avec Docker Compose.

1 Objectif du projet

Ce projet sert de base de test pour :

  • stocker des embeddings de texte dans PostgreSQL via pgvector ;
  • effectuer une recherche par similarité sur ces embeddings ;
  • exposer une API HTTP simple pour interroger la base.

Workflow

1

Démarrer l’environnement

À la racine du projet. Lance la base et l’API.

docker compose up --build
2

Insérer des documents

Dans un autre terminal. Insère 5 docs et affiche le plus proche.

docker compose exec api python scripts/demo.py
Requête personnalisée : 
docker compose exec -e DEMO_QUERY="..." api python scripts/demo.py
3

Tester l’API

Retourne le document le plus similaire au texte envoyé.

curl -X POST "http://localhost:8001/search" \
  -H "Content-Type: application/json" \
  -d '{"query": "Votre question ici"}'

5 Embeddings

Fonction centrale : get_embedding_for_text(text) dans app/vector_utils.py.

Texte → vecteur 1536 dimensions
Avec clé OpenAI
OPENAI_API_KEY défini → modèle text-embedding-3-small
Sans clé
Vecteur déterministe (hash du texte), pour tester sans API
Recherche en base : opérateur <-> (distance euclidienne) sur le type vector de PostgreSQL.

6 Arrêt et nettoyage

Arrêter les services : docker compose down

Supprimer aussi les données : docker compose down -v
